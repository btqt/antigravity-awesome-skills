# Các Nguyên tắc Tài liệu API

> Tài liệu tốt = nhà phát triển vui vẻ = API được đón nhận.

## Những điều Cần thiết OpenAPI/Swagger

```
Bao gồm:
├── Tất cả các điểm cuối với ví dụ
├── Lược đồ yêu cầu/phản hồi
├── Yêu cầu xác thực
├── Định dạng phản hồi lỗi
└── Thông tin giới hạn tốc độ
```

## Tài liệu Tốt Có

```
Những điều cần thiết:
├── Bắt đầu nhanh / Hướng dẫn
├── Hướng dẫn xác thực
├── Tham chiếu API đầy đủ
├── Hướng dẫn xử lý lỗi
├── Ví dụ mã (nhiều ngôn ngữ)
└── Nhật ký thay đổi
```
