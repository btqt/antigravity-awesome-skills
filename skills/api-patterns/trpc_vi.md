# Các Nguyên tắc tRPC

> An toàn kiểu End-to-end cho các monorepo TypeScript.

## Khi nào nên sử dụng

```
✅ Rất phù hợp:
├── TypeScript ở cả hai đầu
├── Cấu trúc Monorepo
├── Các công cụ nội bộ
├── Phát triển nhanh
└── An toàn kiểu quan trọng

❌ Ít phù hợp:
├── Khách hàng không dùng TypeScript
├── API Công khai
├── Cần các quy ước REST
└── Backend đa ngôn ngữ
```

## Lợi ích Chính

```
Tại sao tRPC:
├── Không cần bảo trì lược đồ
├── Suy diễn kiểu End-to-end
├── Tự động hoàn thành IDE trên toàn bộ stack
├── Thay đổi API phản ánh tức thì
└── Không có bước tạo mã
```

## Các Mẫu Tích hợp

```
Các thiết lập phổ biến:
├── Next.js + tRPC (phổ biến nhất)
├── Monorepo với các kiểu chia sẻ
├── Remix + tRPC
└── Bất kỳ frontend + backend TS nào
```
